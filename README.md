# GosqlDB: Автономная и отказоустойчивая СУБД на Go для изолированных и критически важных систем.

## Назначение и Область Применения

**GosqlDB** представляет собой специализированную систему управления
базами данных (СУБД), разработанную на языке программирования Go с целью
обеспечения суверенитета данных, максимальной отказоустойчивости и
безопасности информации.

Проект предназначен для эксплуатации в критически важных
инфраструктурах.

Архитектура системы исключает использование проприетарных зависимостей,
обеспечивая полный контроль над исходным кодом и бинарными артефактами.

------------------------------------------------------------------------

# Ключевые Архитектурные Особенности

## Гарантированная Целостность Данных (ACID)

Система реализует полный спектр свойств транзакционности:

-   **Атомарность** --- операции выполняются полностью либо не
    выполняются вовсе.
-   **Согласованность** --- данные переходят только между корректными
    состояниями.
-   **Изоляция** --- параллельные транзакции не влияют друг на друга
    благодаря MVCC.
-   **Долговечность** --- подтвержденные данные сохраняются даже при
    аварийном отключении питания благодаря журналу WAL.

------------------------------------------------------------------------

## Архитектура Хранения LSM-Tree

Для обеспечения высокой скорости записи используется архитектура
Log-Structured Merge-tree:

### MemTable

Буферизация записей в оперативной памяти с использованием
структурированных деревьев.

### WAL (Write-Ahead Log)

Журналирование всех изменений перед фиксацией в памяти.

### SSTable

Сериализация данных на диск в отсортированном виде.

### Compaction

Фоновый процесс слияния и оптимизации файлов хранения.

------------------------------------------------------------------------

# Безопасность и Изоляция

-   Использование пакета **internal** языка Go предотвращает доступ к
    ядру системы извне.
-   Минимизация пауз сборщика мусора через использование пулов объектов.
-   Стандартизированная система именования для повышения читаемости и
    аудита кода.

# Структура Проекта

	gosql-db/
	├── cmd/
	│   └── gosqld/                 # Точка входа (бинарный файл сервера)
	│       ├── main.go             # Инициализация, запуск сервера, обработка сигналов ✅
	│       └── config.go           # Парсинг конфигурационного файла (YAML/TOML) ✅ 
	├── pkg/                        # Публичные пакеты библиотеки ✅
	│   └── client/                 # Клиентская библиотека для подключения к СУБД
	├── internal/                   # Внутренняя реализация (не доступна извне)
	│   ├── storage/                # Подсистема хранения данных (Storage Engine)
	│   │   ├── wal/                # Write-Ahead Log (Журнал предзаписи)
	│   │   │   ├── log.go          # Логика записи сегментов WAL ✅
	│   │   │   ├── reader.go       # Чтение и восстановление из WAL
	│   │   │   └── segment.go      # Управление файлами сегментов ✅
	│   │   ├── memtable/           # Структура данных в памяти (MemTable)
	│   │   │   ├── table.go        # Реализация на основе ART или SkipList
	│   │   │   └── iterator.go     # Итератор по данным в памяти
	│   │   ├── sstable/            # Формат файлов на диске (SSTable)
	│   │   │   ├── writer.go       # Сериализация и запись SSTable
	│   │   │   ├── reader.go       # Чтение и поиск в SSTable (с использованием Bloom Filter)
	│   │   │   └── block.go        # Структура блоков данных внутри файла
	│   │   ├── compaction/         # Механизм слияния и очистки (Compaction)
	│   │   │   ├── strategy.go     # Стратегии слияния (Level, Size-Tiered)
	│   │   │   └── worker.go       # Фоновые горутины для компактификации
	│   │   ├── index/              # Индексные структуры (B-Tree для вторичных ключей)
	│   │   │   ├── btree.go        # Реализация B+Tree
	│   │   │   └── manager.go      # Управление жизненным циклом индексов
	│   │   └── engine.go           # Главный интерфейс движка: Put, Get, Delete, Scan ✅
	│   │
	│   ├── transaction/            # Менеджер транзакций (ACID)
	│   │   ├── txn.go              # Объект транзакции (начало, коммит, откат) ✅
	│   │   ├── mvcc.go             # Реализация Multi-Version Concurrency Control
	│   │   ├── lock.go             # Управление блокировками (если нужно для SSI)
	│   │   └── snapshot.go         # Создание снимков данных для изоляции
	│   │
	│   ├── sql/                    # Обработка SQL-запросов
	│   │   ├── parser/             # Парсер SQL (обертка над vitess или свой рекурсивный спуск)
	│   │   │   ├── lexer.go        # Лексический анализ ✅
	│   │   │   ├── parser.go       # Синтаксический анализ -> AST ✅ 
	│   │   │   └── ast.go          # Определение узлов Абстрактного Синтаксического Дерева ✅ 
	│   │   ├── planner/            # Планировщик запросов
	│   │   │   ├── optimizer.go    # Оптимизатор (выбор плана выполнения)
	│   │   │   ├── stats.go        # Сбор и использование статистики таблиц
	│   │   │   └── plan.go         # Представление плана выполнения (Plan Node)
	│   │   └── executor/           # Исполнитель запросов
	│   │       ├── exec.go         # Главный цикл исполнения
	│   │       ├── operators/      # Операторы: Scan, Filter, Join, Aggregate, Project
	│   │       └── context.go      # Контекст выполнения запроса
	│   │
	│   ├── network/                # Сетевой слой
	│   │   ├── server.go           # TCP Listener, прием соединений ✅
	│   │   ├── protocol/           # Бинарный протокол взаимодействия (клиент-сервер)
	│   │   │   ├── message.go      # Форматы сообщений (Query, Result, Error)
	│   │   │   └── codec.go        # Кодирование/декодирование (Binary/JSON)
	│   │   └── handler.go          # Обработчик входящих запросов (маршрутизация к SQL двигателю) ✅
	│   │
	│   ├── meta/                   # Метаданные и системный каталог
	│   │   ├── schema.go           # Хранение схем таблиц, колонок, типов данных
	│   │   └── catalog.go          # Системные таблицы (pg_catalog аналог)
	│   │
	│   └── util/                   # Утилиты
	│       ├── pool.go             # sync.Pool для буферов и объектов
	│       ├── arena.go            # Управление Memory Arenas (Go 1.20+)
	│       ├── fsync.go            # Абстракции над системными вызовами fsync
	│       └── errors.go           # Типизированные ошибки проекта ✅
	│
	├── testdata/                   # Данные для интеграционных тестов
	├── scripts/                    # Скрипты сборки, бенчмарков, генерации кода
	├── Makefile                    # Автоматизация сборки и тестирования
	├── go.mod                      # Модуль Go

# Инструкция по Сборке

## 1. Инициализация зависимостей

``` bash
go mod tidy
go mod verify
```

## 2. Компиляция

Создание оптимизированного бинарного файла:

``` bash
go build -ldflags="-s -w" -o gosqld ./cmd/gosqld
```

------------------------------------------------------------------------

# Конфигурация

Создайте файл `config.yaml`.

Пример минимальной конфигурации:

``` yaml
server:
  host: "127.0.0.1"
  port: 5432

storage:
  data_dir: "./data/db"
  wal_dir: "./data/wal"

logging:
  level: "info"
```

------------------------------------------------------------------------

# Запуск

Запуск сервера:

``` bash
./gosqld --config config.yaml
```

Режим разработки:

``` bash
go run ./cmd/gosqld --config config.yaml
```

# Статус Разработки

Проект находится в стадии активной разработки.

На текущий момент реализованы:

-   базовый движок хранения данных;
-   механизм журналирования WAL;
-   первичная система транзакций;
-   сетевой интерфейс взаимодействия.

Планируется внедрение:

-   расширенного оптимизатора SQL-запросов;
-   криптографического шифрования данных на уровне диска;
-   механизма распределенного кластера.
